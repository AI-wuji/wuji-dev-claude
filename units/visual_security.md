# 视觉作战师

## UI 设计审查流程

```
检测到 UI 任务
    ↓
自动激活 impeccable（静默）
    ↓
1. impeccable teach → 了解项目背景
2. impeccable shape → 确定设计方向
3. 执行针对性审查命令（audit/critique/polish/distill/animate/colorize/typeset/arrange）
4. 应用八大反模式自查
    ↓
交付（含设计决策说明）
```

## 反模式快速清单

| # | 反模式 | 症状 | 修复 |
|---|--------|------|------|
| 1 | AI塑料渐变 | 紫橙渐变、彩虹渐变 | HSL 精确定义渐变，克制用色 |
| 2 | 默认字体 | Inter/Roboto/Arial | Clash Display / Satoshi / Outfit / Geist / Noto Sans SC |
| 3 | 灰色地狱 | 灰底+灰字+灰色卡片 | 中性色加色调倾向（蓝灰/暖灰），拉开明度差 |
| 4 | 卡片套娃 | 卡片里嵌套卡片 | 间距、分割线、背景色差区分层级 |
| 5 | 纯黑纯白 | #000 / #fff | 暗色 #0a0a0f 系，亮色 #fafaf9 系 |
| 6 | 弹性缓动 | bounce/elastic | cubic-bezier(0.4, 0, 0.2, 1) 平滑减速 |
| 7 | 图标网格 | 千篇一律的功能列表 | 不对称布局，交错的图文关系 |
| 8 | 无层级阴影 | 大而模糊的默认阴影 | 多层阴影叠加，模拟真实光影 |

## 排版系统

### 字体选择
- 标题字体要有性格，正文字体要好读
- 西文：Clash Display、Satoshi、Outfit、Geist、Sentient
- 中文：Noto Sans SC、思源黑体、霞鹜文楷（创意场景）

### 层级规范
- 标题/副标题/正文至少3级清晰可辨
- 相邻层级字号差距 ≥ 4px
- 标题行高 1.1-1.3，正文行高 1.5-1.7
- 正文每行不超过65字符

### 字号参考（16px 基准）

| 层级 | 字号 | 行高 | 用途 |
|------|------|------|------|
| H1 | 36-48px | 1.1 | 页面主标题 |
| H2 | 24-32px | 1.2 | 区块标题 |
| H3 | 18-22px | 1.3 | 子标题 |
| Body | 15-16px | 1.6 | 正文 |
| Small | 13-14px | 1.5 | 辅助文字 |

## 色彩系统

- 用 OKLCH 或 HSL 定义颜色（方便调色和一致性）
- 中性色要有色调倾向，不要纯灰
- 60-30-10 法则：主色60%、辅助色30%、强调色10%

### 中性色示范（暖灰为例）

```
背景: hsl(40, 10%, 98%)    不是 #fff
卡片: hsl(40, 8%, 94%)     不是 #f5f5f5
边框: hsl(40, 6%, 85%)     不是 #ddd
文字主: hsl(40, 5%, 15%)   不是 #111
文字辅: hsl(40, 4%, 40%)   不是 #666
```

## 间距与布局

基准网格：4px

```
xs:  8px  — 紧密关联的元素
sm:  12px — 相关元素
md:  16px — 标准间距
lg:  24px — 区块内间距
xl:  32px — 小节间距
2xl: 48px — 大区块间距
3xl: 64px — 页面章节间距
4xl: 96px — 页面级间距
```

- 不卡片套卡片，用间距和背景色区分层级
- 不对称布局比对称网格更有视觉张力
- 内容宽度控制在 640-720px（阅读舒适区）

## 动效规范

| 类型 | 时长 | 缓动 |
|------|------|------|
| 微交互(hover, 点击) | 150-250ms | cubic-bezier(0.4,0,0.2,1) |
| 小过渡(展开, 切换) | 200-350ms | 同上 |
| 页面切换 | 300-500ms | 同上 |

- 只用 `transform` 和 `opacity` 做动画（GPU加速）
- 尊重 `prefers-reduced-motion`

## 交付前自检

```
□ 避坑：有没有触犯八大反模式？
□ 排版：字体有性格吗？层级分明吗？
□ 色彩：避开纯黑纯白了吗？灰色有倾向吗？
□ 间距：都是4的倍数吗？节奏感好吗？
□ 动效：流畅自然吗？有没有奇怪弹跳？
□ 层级：视觉层级清晰吗？一眼看到重点？
□ 干净：能去掉的多余元素去掉了没？
□ 响应：移动端/平板/桌面都正常吗？
□ 可访问：对比度≥4.5:1吗？键盘能用吗？
```

# 安全局 SOP

## 进攻（反向工程）

| 目标 | 方法 | 注意 |
|------|------|------|
| Python 源码 | 直接读源码 | 检查 LICENSE |
| .pyc 编译 | uncompyle6 / decompyle3 | Python 3.9+ 可能不完整 |
| Electron (asar) | `npx asar extract app.asar dest/` | 最易破解的格式 |
| .NET (C#) | dnSpy | 还原度极高 |
| .NET 加壳 | de4dot → dnSpy | 先脱壳 |
| PyInstaller | pyinstxtractor → uncompyle6 | 需修复 magic number |
| UPX 加壳 | `upx -d` | 直接解压 |
| C/C++ 原生 | Ghidra / IDA | 伪代码级，难度较高 |

### 主动分析（沙盒环境）
- Process Monitor：监控文件/注册表/网络活动
- API Monitor：拦截 Windows API 调用
- Wireshark / Fiddler：抓包分析网络通信
- x64dbg：动态调试（仅限授权测试）

## 防御（保护方案）

| 级别 | 措施 | 难度 | 适用场景 |
|------|------|------|---------|
| L1 | asar 加密 + UPX 压缩壳 | ★ | 基础保护 |
| L2 | 代码混淆 (JS/Python) | ★★ | 一般商业软件 |
| L3 | 关键逻辑 Native 化 | ★★★ | 重要商业软件 |
| L4 | 反调试 + 完整性校验 | ★★★★ | 高价值软件 |
| L5 | VMProtect/Themida | ★★★★★ | 核心知识产权 |
| L5+ | Tauri/Rust 原生编译 | ★★★★★ | 终极方案 |

### 建议
- 没有绝对安全，目标是提高破解成本 > 重新开发的成本
- 普通产品 L1+L2 足够了
- 多层防护比单层强得多

## 安全红线

```
❌ 不直接复制受版权保护的代码
❌ 提取功能后要重写实现（不能改几个变量名就过）
❌ 注意原项目的许可证要求
❌ 逆向工程结果仅用于学习/兼容目的
❌ 不用于商业侵权的反向工程
```

## 打靶场安全检查

执行前跑一次扫描：
```
python scripts/target_range.py code <待执行的代码文件>
python scripts/target_range.py permission "<操作描述>"
```

### 扫描维度

| 扫描类型 | 检查内容 |
|---------|---------|
| code | 命令注入、动态执行(eval/exec)、硬编码密钥、危险导入 |
| plugin | ComfyUI插件完整性、依赖数量、代码安全 |
| dependency | 依赖名安全性、可疑关键字检测 |
| config | 敏感配置(key/password/token) |
| permission | 操作风险等级评估(G/O/R三级) |

### 评分标准

```
≥80: ✅ 通过，可以执行
60-79: ⚠️ 有条件通过，处理警告后执行
<60: ❌ 未通过，必须修复失败项
```
